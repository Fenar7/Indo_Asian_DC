## Project Overview

This repository is a Next.js frontend project initialized directly in the root folder.

Current stack:

- Next.js App Router
- React 19
- TypeScript
- Tailwind CSS v4
- SCSS for global and route-level styling
- Embedded Sanity Studio via `next-sanity`

The project is still being built in a static-first way. The current priority has been high-fidelity UI implementation from Figma and screenshot references before wiring any real data, cart logic, auth, or CMS-driven rendering.

## Global Styling System

### Fonts

The project uses SF Pro globally.

Font source:

- `public/fonts/SFPro`

Font setup lives in:

- `src/app/globals.scss`

It is available:

- as the default global app font
- in SCSS via `$font-sf-pro`
- via utility usage through `font-sans` and `font-sf-pro`

### Colors

Shared design tokens are defined in:

- `src/styles/_tokens.scss`

Current token values:

- `$pure-black: #000`
- `$pure-white: #fff`
- `$hash-grey-2: #767676`
- `$cloud-grey: #e3e5e6`
- `$main-cta-red: #dc2b1b`
- `$stroke-grey: #dedede`

These are also exposed through global CSS variables and utility-friendly names.

Examples:

- `bg-main-cta-red`
- `text-hash-grey-2`
- `border-stroke-grey`
- `bg-cloud-grey`
- `text-pure-white`

### Sass Injection

Shared Sass tokens are auto-injected into every `.scss` file through:

- `next.config.ts`

Current setup:

```ts
sassOptions: {
  additionalData: '@use "@/styles/tokens" as *;',
}
```

This means route SCSS files can use the global variables without manually importing the tokens file.

## Reusable Component Direction

The project is moving toward reusable static-first screen composition.

Current approach:

- route files stay thin
- route-level SCSS handles page-specific layout and fidelity
- screen/component files hold reusable UI blocks
- larger page sections are intentionally structured so they can later become dynamic without rewriting the layout from scratch

This pattern is already in use in:

- `src/components/shop-page/ShopPageScreen.tsx`
- `src/components/product-detail/ProductDetailScreen.tsx`
- `src/components/cart-page/CartPageScreen.tsx`

## Current Conventions Being Followed

- keep the route-level container structure the user asked for
- use the Tailwind `container` wrapper pattern already established in the project
- avoid adding arbitrary outer route padding unless the design needs it
- use SCSS for exact route styling
- use only the globally defined fonts and color tokens
- build static UI first, then wire functionality later
- keep pages responsive from the start
