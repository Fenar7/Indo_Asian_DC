## Project Overview

This repository is a Next.js frontend project initialized in the root of the current folder.

Current stack:

- Next.js App Router
- React 19
- TypeScript
- Tailwind CSS v4
- SCSS for route-level and global styling
- Embedded Sanity Studio via `next-sanity`

The project is still in static-first mode. The main goal so far has been to implement high-fidelity UI screens from Figma and screenshot references before making anything dynamic.

## Global Styling System

### Fonts

SF Pro Display is the global font family.

Font source:

- `public/fonts/SFPro`

Font loading and setup live in:

- `src/app/globals.scss`

Available globally:

- as the base app font
- in SCSS via `$font-sf-pro`
- in utility usage through `font-sans` and `font-sf-pro`

### Colors

Global shared SCSS tokens are defined in:

- `src/styles/_tokens.scss`

Current token values:

- `$pure-black: #000`
- `$pure-white: #fff`
- `$hash-grey-2: #767676`
- `$cloud-grey: #e3e5e6`
- `$main-cta-red: #dc2b1b`
- `$stroke-grey: #dedede`

These are also exposed in global CSS variables and mapped to utility-friendly names.

Examples:

- `bg-main-cta-red`
- `text-hash-grey-2`
- `border-stroke-grey`
- `bg-cloud-grey`
- `text-pure-white`

### Sass Injection

Shared Sass tokens are automatically injected into every `.scss` file through:

- `next.config.ts`

That means route SCSS files do not need to manually import the tokens file.

## Current Conventions Being Followed

- Keep route-level wrapper structure minimal
- Use the Tailwind `container` shell the user requested
- Keep custom layout/pixel detail in SCSS
- Use global SF Pro and global color tokens everywhere
- Build static UI first
- Add functionality only after visual fidelity is locked
- Keep pages responsive from the beginning
- Prefer route screen components for larger pages
