## 3. Application Architecture

### Pages (`src/app/`)

| Route | File | Type | Purpose |
|-------|------|------|---------|
| `/` | `page.tsx` | Server Component | Shop home — hero, category chips, filter bar, sidebar, product grid, cart panel |
| `/login` | `login/page.tsx` | Client Component | Password-protected site entry |
| `/product/[id]` | `product/[id]/page.tsx` | Server Component | Product detail page with related products |
| `/cart-page` | *(exists)* | Client Component | Full cart/checkout page |
| `/studio` | `studio/[[...index]]/page.tsx` | — | Embedded Sanity Studio |

### Key Components

#### `src/components/shop-page/ShopPageScreen.tsx` (1,064 lines)
- **Type:** Client Component (`"use client"`)
- **Props:** `heroSlides`, `categories`, `products`
- **Features:**
  - Header with brand logo, search bar, cart button
  - Hero image slider (auto-rotating)
  - Category chip row (horizontal scroll, sticky)
  - Filter bar — Weight, Tag (badge), Unit, Price Range pills
  - **Sidebar** — vertical sticky category list with counts
  - **Product grid** — grid/list view toggle, ProductCards
  - **Cart panel** — right-side sliding cart with quantity controls
  - **Search dropdown** — uses `createPortal` to render above sticky header

#### `src/components/product-detail/ProductDetailScreen.tsx`
- Product detail view with image gallery, info, add-to-cart
- Related products carousel

#### `src/context/CartContext.tsx`
- **Type:** Client Context
- **Storage:** `localStorage` key = `"indo_asian_cart"`
- **Hydration-safe:** Reads localStorage synchronously in state initializer
- **Actions:** `addToCart`, `removeFromCart`, `updateQuantity`, `clearCart`

### Server Data Flow

```
page.tsx (Server)
  ├─ heroQuery → HeroSlide[]
  ├─ categoriesQuery → SanityCategory[] (sorted by order asc)
  └─ productsQuery → SanityProduct[] (with category._ref as categoryId)
       ↓
ShopPageScreen (Client)
       ↓
  Filtering pipeline:
    1. Category filter (activeCategoryId)
    2. Search query (name, code, unit, weight)
    3. Weight filter (OR within selected set)
    4. Tag/Badge filter (OR within selected set)
    5. Unit filter (OR within selected set)
    6. Price range filter
```
